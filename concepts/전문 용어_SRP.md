# SRP(Single Responsibility Principle, 단일 책임 원칙)

_백엔드 개발자라면 반드시 몸에 익혀야 하는 핵심 개념이다._

특히 Node.js + Express + Prisma 같은 구조에서는 코드 유지보수성과 협업 효율을 좌우한다.

**“하나의 파일(또는 클래스, 함수)은 하나의 일만 해야 한다.”**
\_ 컨트롤러는 “요청을 받고 응답을 만드는 일”만,
\_ 서비스는 “비즈니스 로직(계산·검증 등)”만,
\_ 모델은 “DB와의 통신”만,
\_ 미들웨어는 “요청을 가공하거나 검증”만

**역할이 딱딱 분리돼서 나중에 DB 구조가 바뀌거나 CDN 주소가 달라져도 다른 부분엔 영향이 거의 없다.**
